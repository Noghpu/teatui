# Enter on Menu force-syncs the head field

## Goal
Make pressing `Enter` on the Generate-screen Menu pane unconditionally write the selected revset into `form.head` (and `form.branch_name`), even when the user has already edited those fields. Arrow-key navigation through the Menu keeps the current dirty-respecting behavior so the user does not lose in-flight edits while scrolling.

## Context
`select_revset` in `src/app.rs` (around line 789) calls `sync_head_from_selected_revset`. That sync method at `src/generate.rs:1267` only writes to `form.head` when `form.head.dirty() == false`. As soon as the user has touched the head field, pressing `Enter` on the Menu silently no-ops on the head â€” that is the reported "head does not reflect the left-pane selection" bug. The expected mental model is: arrows preview, `Enter` commits. This ticket is intentionally tiny and independent of the head-as-change-id rework so it can land first.

## Non-Goals
- Do not change how arrow navigation syncs head â€” it must still skip dirty fields.
- Do not change what string is written into the head field. The label format is the responsibility of `head-field-as-change-id`.
- Do not change picker option contents.
- Do not introduce a new action; reuse the existing `select_revset` entry point.

## Design Decisions
- Add `GenerateState::force_sync_head_from_selected_revset(&mut self)` that:
  - Writes `selected_revset().label()` into `form.head` regardless of `dirty()`.
  - Writes `selected_revset().bookmarks().first().cloned().unwrap_or_default()` into `form.branch_name` regardless of `dirty()`.
  - Clears the `dirty` flag on both fields after writing (the value now matches the freshly-committed selection, so subsequent arrow-key syncs work again).
  - Calls `self.validate_form()`.
- Keep `sync_head_from_selected_revset` exactly as is for arrow-key navigation.
- `select_revset` in `src/app.rs:789` calls the new force variant instead of the dirty-respecting one.
- No public API removal â€” both methods are public.

## Implementation Plan
1. In `src/generate.rs`, add `force_sync_head_from_selected_revset(&mut self)` next to `sync_head_from_selected_revset`. Share the local helpers for picking branch name.
2. After writing each field, reset its `dirty` flag. The right way: re-establish `initial == value` semantics for both fields. Inspect `TextFieldState` (`src/generate.rs:154-198`) and `FieldState`/`PickerFieldState`. If there is no direct API to clear `dirty`, add a small `pub fn mark_clean(&mut self)` that sets `initial.clone_from(&value)` and `dirty = false`. Apply to the head and branch fields only.
3. In `src/app.rs`, change `select_revset` (around line 789) to call `self.generate.force_sync_head_from_selected_revset()`.
4. Add a unit test in `src/generate.rs`: `select_revset_overwrites_head_even_when_dirty`. Build a `GenerateState` with at least two revsets, mark the head field as dirty (`form.head.set_value("user-edited")`), call `force_sync_head_from_selected_revset`, assert `form.head.display_value()` matches the currently-selected revset's label.
